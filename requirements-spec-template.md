# 要求仕様書テンプレート

> **記述の基本原則**
> - 機能要求は「**[条件] のとき、[主語] は [動作] する**」の形で書く
> - 仕様はテスト可能（合否判定できる）な粒度まで分解する
> - 「〜できる」（能力）と「〜する」（動作）を使い分ける
> - 非機能要求は必ず数値・基準で表現する

### UC-01: [ユースケース名]

| 項目 | 内容 |
|------|------|
| アクター | |
| 事前条件 | このユースケースが開始できる状態 |
| 事後条件（正常） | 正常終了したときのシステム状態 |
| 事後条件（異常） | 異常終了したときのシステム状態 |

**基本フロー**

1. [アクター] が〜する
2. システムが〜を表示する
3. [アクター] が〜を入力する
4. システムが〜を保存し、〜を表示する

**代替フロー**

- 3a. 入力値が不正な場合 → システムはエラーメッセージ「〜」を表示し、3 に戻る
- 4a. 保存に失敗した場合 → システムはエラーを記録し、ユーザーに「〜」を通知する
